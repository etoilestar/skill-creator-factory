# Scientific Report Writer

## 输入要求

用户需要提供主题、研究领域、文章结构（如引言、方法、结果、讨论等）。

## 工作流程

1. 用户提供科研报告的主题和结构要求。
2. 调度模型解析输入，并检查信息的完整性。
3. 调用 qwen3:32b 模型的 API 接口，生成科研报告。
4. 格式化结果，确保段落首行缩进两个字。
5. 输出最终的科研报告文本。

## 能力调度说明

1. 当前调度模型负责理解用户需求、检查缺失字段、组织执行流程。
2. 使用 OpenAI 兼容接口调用 qwen3:32b 模型生成科研报告正文。

## 资源使用说明

- 调用外部接口时，将请求发送到 http://172.18.127.67:11434/v1/，获取生成的科研报告文本。

## 输出要求

生成的科研报告应为文本格式，段落首行缩进两个字。

## 依赖与配置

无。

## 失败处理

如果缺少关键信息，将提示用户补充必要的输入。若外部模型不可用，则输出错误信息，并建议用户检查模型的可用性。
